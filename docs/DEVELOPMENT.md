# Development

## Commands

```powershell
dotnet build
dotnet test tests\PinyinHelper.Tests\PinyinHelper.Tests.csproj
.\build.ps1
.\release.ps1
.\install-plugin.ps1
```

## Release Package

`release.ps1` reads `plugin.json` and creates:

```text
dist\PinyinHelper-v0.2.0.zip
```

The zip contains the `PinyinHelper` folder and only runtime files:

```text
PinyinHelper/Community.PowerToys.Run.Plugin.PinyinHelper.dll
PinyinHelper/Community.PowerToys.Run.Plugin.PinyinHelper.deps.json
PinyinHelper/PinYinConverterCore.dll
PinyinHelper/plugin.json
PinyinHelper/Images/PinyinHelper.dark.png
PinyinHelper/Images/PinyinHelper.light.png
```

Debug symbols are intentionally excluded.

## Project Notes

- Target framework: `net9.0-windows10.0.22621.0`.
- PowerToys Run references come from `C:\Program Files\PowerToys`.
- Pinyin data comes from `PinYinConverterCore`, using the compatible `Microsoft.International.Converters.PinYinConverter` namespace/API.
- `Assets\PowerToysIcon.png` is generated by `PrepareIcon.ps1` from `Assets\PinyinHelperIcon.png` and is gitignored.
- PowerToys Run integration is in `Main.cs`.
- Chinese lookup is in `PinyinLookupService.cs`.
- English-mode Zhuyin keyboard parsing is in `ZhuyinKeyboardDecoder.cs`.
- Default pinyin-to-character selection is in `PinyinReverseLookupService.cs`.
- Tests live in `tests\PinyinHelper.Tests`.
- Future work lives in `docs\TODO.md`.

## Command Palette

This project is not directly installable as a PowerToys Command Palette extension.
The current host integration is PowerToys Run-specific: `Wox.Plugin.IPlugin`, `plugin.json`, and the PowerToys Run plugin folder.

If Command Palette migration becomes necessary, split the host-independent logic into a core project and add a separate CmdPal adapter:

```text
PinyinHelper.Core
PinyinHelper.PowerToysRun
PinyinHelper.CommandPalette
```
